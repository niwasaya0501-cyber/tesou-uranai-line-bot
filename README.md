# 手相占いWebアプリ

URLを開いて手のひらの写真を送るだけで、OpenAI (gpt-4o-mini Vision) が手相鑑定をし、その場（同じページ）に結果を表示するWebアプリです。LINEは使わず、URLだけで完結します。バックエンドはVercel Serverless Functions。課金が発生するのはOpenAI APIのみで、それ以外は無料枠で構成できます。

## ディレクトリ構成

```
手相占い/
├── api/
│   ├── submit.js          # 「画像＋悩み」を受けて鑑定し、結果をレスポンスで返す
│   ├── chat.js            # 鑑定結果を踏まえた続きの質問（ラリー）に答える
│   └── delete-session.js  # 鑑定内容・やりとりをその場で削除する
├── lib/
│   ├── openai.js          # OpenAI Vision呼び出し・プロンプト生成／ラリー用のテキスト会話
│   ├── image.js           # sharpでのリサイズ・EXIF除去
│   ├── conversation.js    # Upstash Redisでの会話セッション保存（ラリー機能用）
│   └── rateLimit.js       # IPアドレス単位の簡易レート制限（費用の際限ない増加を防ぐ）
├── public/
│   ├── index.html         # 悩み5択＋画像選択＋鑑定結果＋チャットの画面
│   ├── app.js              # 画面の状態管理／送信処理／チャット処理
│   └── style.css
├── .env.example
├── package.json
├── vercel.json
└── README.md
```

## 処理の流れ

1. ユーザーがURLを開く
2. 悩み（恋愛／仕事／人間関係／自己分析／その他）を選択し、写真を選ぶ（「その他」を選ぶと自由入力欄が表示され、具体的な悩みをテキストで入力できる）
3. ブラウザ側で軽くリサイズしてから `/api/submit` にPOST
4. サーバー側でsharpにより最終リサイズ（長辺768px・EXIF除去）→ OpenAI Visionで鑑定 → 結果をレスポンスとして返し、同じページに表示
5. 鑑定結果はセッションとして保存され、結果の下にあるチャット欄からそのまま続けて質問できる（「ラリー」機能。詳細は本README末尾を参照）

ブラウザには`localStorage`に匿名のセッションIDが保存され、同じ端末・ブラウザから開き直しても続きの会話を保持できます（ログインの仕組みはありません）。

## 1. 事前準備

- Node.js（18以上推奨）
- [Vercel](https://vercel.com/) アカウント（無料枠でOK）
- [OpenAI Platform](https://platform.openai.com/) アカウント・APIキー

## 2. Upstash Redis（会話の一時保存）の作成

鑑定結果のあと続けて質問できる「ラリー」機能と、レート制限のカウントのために、無料のデータストアを用意します。VercelのStorage連携から作成でき、OpenAI以外は引き続き無料枠内に収まります。

1. Vercelダッシュボードの左メニュー（プロジェクトを開く前の、チーム全体の画面）から **`ストレージ`** を開く（プロジェクトの中に入っている場合は、左メニューの **`ストレージ`** からでもOK）
2. **`Create Database`**（データベースを作成）→ **`Upstash`** → **`Redis`** を選択
3. 無料プラン（Free）のまま、リージョンなどはデフォルトで作成
4. 作成後、対象のVercelプロジェクトに **`Connect`**（接続）する
5. 接続すると、`KV_REST_API_URL` と `KV_REST_API_TOKEN` という2つの環境変数が、そのプロジェクトに自動で追加されます（手動でコピーする必要はありません）
6. ローカルでも動作確認したい場合は、Redisデータベースの画面にある「.env」タブ等からこの2つの値を確認し、ローカルの`.env`にも追記してください

## 3. Vercelへのデプロイ

### 3-1. 初回デプロイ

```bash
npm install -g vercel   # 未インストールの場合
cd 手相占い
vercel login
vercel                  # プロジェクトをリンク（初回は質問に答えて設定）
```

### 3-2. 環境変数の設定

Vercelダッシュボード（Project Settings > Environment Variables）、またはCLIで設定します。

```bash
vercel env add OPENAI_API_KEY
```

（`KV_REST_API_URL` と `KV_REST_API_TOKEN` は、2章でVercelプロジェクトに接続していれば自動で追加済みです）

（`.env.example` を参考に、Production / Preview / Development すべてに登録してください）

ローカルで動作確認したい場合は `.env.example` をコピーして `.env` を作成し、実際の値を入れてください（`.env` は `.gitignore` 済みでコミットされません）。

### 3-3. 本番デプロイ

```bash
vercel --prod
```

デプロイ完了後に表示されるURL（例：`https://your-app.vercel.app`）が、そのままユーザーに共有するURLになります。

## 4. 動作確認

1. デプロイされたURLをブラウザで開く
2. 悩みを選択 → 手のひらの写真を選択 →「この写真で占う」をタップ
3. 「鑑定中です…」→ 数秒〜数十秒後に、そのページに鑑定結果が表示されることを確認
4. 結果の下のチャット欄に何か質問を送ると、鑑定結果を踏まえた返信が届くことを確認（ラリー機能）

## 5. 環境変数一覧

| 変数名 | 説明 | 確認場所 |
|---|---|---|
| `OPENAI_API_KEY` | OpenAI APIキー | OpenAI Platform > API keys |
| `KV_REST_API_URL` | 会話セッション保存・レート制限用RedisのURL | Vercel > Storage > 作成したUpstash Redis |
| `KV_REST_API_TOKEN` | 同上のアクセストークン | Vercel > Storage > 作成したUpstash Redis |

## 6. コストに関する注意

- 課金が発生するのはOpenAI APIのみです。Vercel・Upstash Redisはいずれも無料枠内で運用できます
- 画像はサーバー側で長辺768pxにリサイズしてからOpenAIに送るため、1回あたりのAPIコストを抑えています
- `max_tokens`（`lib/openai.js`内）で出力上限を設定し、応答の暴走・コスト増加を防いでいます
- 両手（2枚）で鑑定した場合、画像1枚あたりのコストが単純計算で2倍になりますが、gpt-4o-miniは非常に低価格なため、1回あたりの負担は引き続き小さいです
- ラリー（鑑定後のチャットでの続きの会話）は**テキストのみ**で行い、画像は再送しないため、1往復あたりのコストは非常に小さいです。1鑑定あたり最大10往復（`lib/conversation.js`の`MAX_TURNS`）で自動的に打ち切り、際限のないコスト増加を防いでいます
- **URLを直接公開する構成では、LINEの友だち登録のような自然な参入障壁がなくなる**ため、`lib/rateLimit.js`でIPアドレス単位の1日あたりの利用回数に上限を設けています（鑑定＝画像を使う呼び出しは1日15回まで、ラリーのテキストのみの返信は1日100回まで。`api/submit.js`・`api/chat.js`内の`DAILY_LIMIT_PER_IP`で変更可能）。上限に達した場合はエラーメッセージが表示され、翌日リセットされます

## 7. トラブルシューティング

| 症状 | 原因 | 対処 |
|---|---|---|
| 鑑定に時間がかかりタイムアウトする | 画像サイズが大きい／Vercelプランの実行時間上限 | `vercel.json`の`maxDuration`を確認し、プランの上限内に収まっているか確認 |
| OpenAIの応答が毎回「もう一度送ってください」になる | 手のひら全体が写っていない、または暗すぎる/ぶれている | 明るい場所で手のひら全体が写るように撮影し直す |
| 「本日の鑑定回数の上限に達しました」と表示される | 同じIPアドレスから1日の利用上限を超えて鑑定した | 翌日になると自動的にリセットされる。上限自体を変えたい場合は`api/submit.js`の`DAILY_LIMIT_PER_IP`を編集 |
| デザイン変更後もページで一瞬だけ古い見た目が表示される | ブラウザ側に古い`style.css`/`app.js`がキャッシュされている | `vercel.json`の`headers`で`index.html`/`app.js`/`style.css`に`no-cache`を設定済み。それでも直らない場合はブラウザのキャッシュを削除して再読み込み |
| チャットで質問しても「404 no active session」のような挙動になる | 鑑定をしていない、またはセッションが24時間で期限切れ・削除済み | 新しく鑑定をやり直す |

## 8. 悩みの選択肢を変更したい場合

`public/index.html` の `.worry-btn` と、`lib/openai.js` / `public/app.js` の `WORRY_LABELS` を同じキーで揃えて編集してください（3箇所とも `key` を一致させる必要があります）。

## 9. ラリー（鑑定後の続きの会話）について

鑑定結果を表示した後、結果の下にあるチャット欄にメッセージを送ると、直前の鑑定結果を踏まえてAIが返信します（`api/chat.js`）。

- 会話は`lib/conversation.js`経由でUpstash Redisに一時保存され、**24時間**で自動的に消えます
- 1回の鑑定につき**最大10往復**（`MAX_TURNS`）まで。それを超えると「このやりとりは一旦ここまで」という案内とともに、消すか残すかを選ぶボタンが表示されます。再度鑑定すれば、また新しく10往復分のラリーができます
- ラリーの返信は**画像を再送しないテキストのみ**のやり取りのため、コストは非常に小さく抑えられています（詳細は「6. コストに関する注意」を参照）
- チャット欄の下にある「やりとりを消す」ボタンから、いつでもその場で鑑定内容・やりとりを削除できます（`api/delete-session.js`）
- セッションは`localStorage`に保存した匿名のセッションIDに紐づいているため、同じ端末・ブラウザのlocalStorageを消去すると、サーバー側に残っていても続きの会話は参照できなくなります
